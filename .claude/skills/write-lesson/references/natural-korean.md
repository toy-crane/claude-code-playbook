# 자연스러운 한국어 (Natural Korean)

LLM 이 생성한 한국어에서 반복 발생하는 **번역투 · AI 냄새 · 의인화 과다** 를 줄이고, 강의 본문의 합니다체 prose 가 한국어 원어민이 쓴 것처럼 읽히게 하기 위한 참조 문서.

**사용법** — write-lesson SKILL.md 의 Step 3d (자기 점검) 에서 체크리스트 D 로 본문을 훑고, 매칭되는 표현을 각 섹션 (A / B / C) 의 대안으로 교체한다.

---

## 세 가지 실패 양식

1. **번역투** — 영어 구조를 직역해 조사·어순이 어색해진 문장
2. **AI 냄새** — 장식·반복·쉼표 남발로 "AI 가 쓴 것" 이 드러나는 문장
3. **의인화 동사 과다** — 도구·시스템을 사람처럼 묘사하는 동사 중첩 (강의 도메인 특유)

---

## A) 번역투 12 패턴

강의 본문에서 자주 등장하는 순으로 정리. 각 항목은 `Before → After` + 한 줄 이유.

### 1. 피동 남용

영어 수동태 직역. 능동으로 전환.

| Before | After |
|---|---|
| 박물관은 평창에 **위치하고 있습니다** | 박물관은 평창에 **있습니다** |
| 재단**에 의해 운영**되던 극장 | 재단**이 운영**하던 극장 |
| 파일이 Claude **에 의해 읽힙니다** | Claude **가** 파일**을 읽습니다** |

### 2. 가지다 (have 직역)

영어 have / has 구문 직역. "있다 / ~이다" 로.

| Before | After |
|---|---|
| 아들을 하나 **가지고 있다** | 아들이 **하나다** |
| 의미를 **가지고 있습니다** | 뜻**입니다** |
| 두 가지 장점을 **가집니다** | 장점이 두 가지**입니다** |

### 3. ~을 통해

영어 through 직역. `~해서 / ~로` 가 자연스러움.

| Before | After |
|---|---|
| 공부**를 통해** 합격했다 | 공부**해서** 합격했다 |
| 대화**를 통해** 요구사항을 정리합니다 | 대화**로** 요구사항을 정리합니다 |

### 4. ~에 대하여 / ~에 대한

영어 about 직역. 관형절 대신 직접 목적어.

| Before | After |
|---|---|
| 과거형**에 대한** 문법 | 과거형 문법 |
| 저**에 대해** 잘 모릅니다 | 저를 잘 모릅니다 |
| Skill**에 대한** 원리 | Skill 의 원리 |

### 5. ~하는 중이다

영어 진행형 -ing 직역. 한국어는 현재형이 자연스러움.

| Before | After |
|---|---|
| 민원을 담당**하는 중이다** | 민원을 담당**한다** |
| 전철이 도착**하는 중입니다** | 전철이 도착**합니다** |

### 6. ~하기 위해

영어 to 부정사 직역. 관계절로 재구성 또는 `~하려면`.

| Before | After |
|---|---|
| 대체**하기 위해** 대기하고 있었다 | 대체**할** 사람들이 대기하고 있었다 |
| 성공**하기 위해서는** 노력이 필요하다 | 성공**하려면** 노력이 필요하다 |

### 7. ~에 있어

영어 in + 명사 직역. 대부분 삭제 가능.

| Before | After |
|---|---|
| 높이는 데**에 있어** 메모리 용량은 | 높이는 **데** 메모리 용량은 |
| 중요함**에 있어** | 중요함**에** |

### 8. '의' 남용

복합어에서 불필요한 조사.

| Before | After |
|---|---|
| 회사**의** 초대 회장 | 회사 초대 회장 |
| 팀**의** 코드 컨벤션 | 팀 코드 컨벤션 |

### 9. '들' 남용

한국어는 복수 구별이 느슨. 문맥으로 복수가 드러나면 '들' 불필요.

| Before | After |
|---|---|
| 많은 학생**들**이 있다 | 많은 학생이 있다 |
| 여러 파일**들**을 읽습니다 | 여러 파일을 읽습니다 |

### 10. 대명사 남발

영어 he / she / it / they 직역. 한국어는 주어 생략이 자연스러움.

| Before | After |
|---|---|
| 철수가 사과를 좋아해서 **그는** 과일가게에서 샀다 | 철수가 사과를 좋아해서 퇴근길에 과일가게에서 샀다 |
| 파일을 읽고 **그것을** 파싱합니다 | 파일을 읽고 파싱합니다 |

### 11. ~로부터

영어 from 직역. `~에서` 로.

| Before | After |
|---|---|
| 한강**으로부터** 10 km | 한강**에서** 10 km |
| 사용자**로부터** 입력을 받습니다 | 사용자**에게** 입력을 받습니다 |

### 12. ~의 경우

영어 in the case of 직역.

| Before | After |
|---|---|
| 저**의 경우에는** | 저는 |
| Skill**의 경우** 자동 호출이 | Skill 은 자동 호출이 |

## B) AI 냄새 7 패턴

LLM 이 생성한 글 특유의 "기계 냄새". 의미상 틀리진 않지만 누적되면 글 전체가 AI 로 드러남.

### 1. 쉼표 남발

한국어 원어민은 절을 이어 쉼표를 많이 쓰지 않음. 영어 복문을 번역해 쉼표가 반복됨. **한 문장에 쉼표 3 개 이상이면 문장 나누기 검토.**

| Before | After |
|---|---|
| 이 구조는 복잡하고, 유지보수가 어렵고, 수정 비용이 커집니다 | 이 구조는 복잡합니다. 유지보수가 어렵고 수정 비용도 큽니다 |

### 2. 한자어 과다

"활용 / 진행 / 제공 / 검토" 같은 한자어가 반복되면 딱딱. 일상어로 교체.

| 한자어 | 일상어 |
|---|---|
| 활용하다 | 쓰다 / 써먹다 |
| 진행하다 | 하다 / 돌리다 |
| 제공하다 | 주다 / 보여주다 |
| 검토하다 | 살펴보다 / 한 번 보다 |
| 적용하다 | 넣다 / 반영하다 |
| 수행하다 | 하다 |

### 3. 모호어 과다

"대체로 / 보통 / 어느 정도 / 다소 / 꽤 / 경우가 많다" 같은 단어가 많으면 주장이 약해짐. 삭제하거나 구체화.

| Before | After |
|---|---|
| 대체로 쉼표를 적게 씁니다 | 쉼표를 적게 씁니다 |
| 어느 정도 도움이 됩니다 | 도움이 됩니다 / 예시 3 중 2 에서 개선됩니다 |

### 4. 무가치 틀

"결론부터 말하면 / 중요한 점은 / 한마디로 / 본질적으로 / 간단히 말해" 같은 도입부는 정보 없음. 삭제.

### 5. 동어 반복 결론

앞 문장이 이미 설명한 내용을 명사형으로 한 번 더 감싸는 꼬리 문장. **삭제가 정답**.

| Before | After |
|---|---|
| …그 조건을 만났을 때만 읽습니다. Progressive Disclosure 3단계가 실제로 동작하는 모습입니다. | …그 조건을 만났을 때만 읽습니다. |

### 6. 문장별 동등 명확성

한 단락 안 모든 문장이 같은 무게로 병렬되면 리듬이 사라짐. **핵심 문장 하나 + 보조 문장** 을 섞어 리듬을 만든다.

### 7. "~해주다" 보조 동사 과용

"처리해줍니다 / 도와줍니다 / 받아줍니다" 가 문단마다 나오면 구어체로 튐. 본동사 단독.

| Before | After |
|---|---|
| 자동으로 처리**해줍니다** | 자동으로 처리**합니다** |
| 파일을 읽어**옵니다** | 파일을 **읽습니다** |

## C) 의인화 동사 회피

도구·시스템을 사람처럼 묘사하는 동사. 한 번은 생동감이지만 반복되면 과한 의인화. **한 문장에 의인화 동사 2 개 이상 겹치면 조기 경보.**

| 의인화 표현 | 대안 |
|---|---|
| 받아주다 | 해소되다 / 정리되다 |
| 끌어내다 | 짚다 / 드러내다 |
| 질문을 던지다 | 질문하다 / 묻다 |
| 답을 주다 | 해결된다 / 풀린다 |
| (도구가) 해준다 | 된다 / 처리한다 |
| 맡아준다 | 담당한다 / 처리한다 |

**예시**:
- Before: "Skill Creator 는 대화로 질문을 **던져** 누락을 **끌어내줍니다**"
- After: "Skill Creator 는 놓치기 쉬운 부분을 질문으로 **짚어줍니다**"

**원인** — LLM 이 "도구가 능동적으로 해준다" 를 강조하려다 의인화 동사를 중첩 사용. 능동성 강조는 **한 동사로 충분**.

## D) 자기 점검 체크리스트

본문 초안을 완성한 후 아래 7 문항으로 훑기. 매칭되면 해당 섹션 (A / B / C) 의 대안으로 교체.

1. 이 문장에서 **도구·시스템이 사람처럼 행동하는 동사** 가 있나? (받아주다 / 끌어내다 / 질문을 던지다 / 해준다)
2. 한 문장에 **보조 동사** ("~주다 / ~버리다 / ~오다") 가 2 개 이상인가?
3. **앞 문장과 같은 말을 명사형으로 감싸는 결론** 이 붙었나? ("~의 모습입니다 / ~의 예입니다")
4. **한자어** 를 쉬운 일상어로 바꿀 수 있나? (활용 / 진행 / 제공 / 검토 / 적용)
5. **"~에 의해 / ~에 대하여 / ~을 통해 / ~에 있어"** 같은 조사 번역투가 있나?
6. **강한 비유** ("못 박다 / 박고") 가 반복되나?
7. 한 문장에 **쉼표가 3 개 이상** 인가? (끊어 쓸 수 있나?)

한 문단에 2 문항 이상 매칭되면 즉시 수정 — 누적되면 글 전체가 AI 냄새.
