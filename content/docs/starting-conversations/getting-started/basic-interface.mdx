## Overview

첫 대화를 시작했는데 터미널에서 하던 감각과 다릅니다. 긴 프롬프트를 여러 줄로 쓰려고 엔터를 눌렀더니 그대로 전송되고, AI가 엉뚱하게 고친 파일을 되돌리고 싶은데 방법을 모릅니다.

이번 레슨은 Claude Code를 매일 쓰며 반드시 마주치는 **세션 관리·특수 입력·권한 통제·되돌리기** 네 축을 실제 동작과 함께 익힙니다.

### 학습 목표

- 세션을 시작·이어받기·종료하고, Context를 비우는 두 방식을 **구별합니다**
- `!`, `@`, `/` 와 단축키로 입력 모드를 빠르게 **전환합니다**
- 권한 모드 6가지와 Auto mode의 안전장치를 **이해하고**, 상황에 맞게 **고릅니다**
- AI의 변경을 Rewind와 Checkpoint로 **되돌립니다**

## 매일 부딪히는 네 가지 인터페이스 축

Claude Code의 입력창은 일반 터미널이 아닙니다. Agent를 움직이는 제어창이고, 그래서 터미널과 다른 규칙이 몇 가지 있습니다.

매일 부딪히는 축은 네 가지입니다.

- **세션** — 한 대화의 시작과 끝, 그리고 Context 관리
- **특수 입력** — `!`, `@`, `/` 세 가지 접두어로 모드 전환
- **권한** — AI가 파일·명령어를 실행하기 전 승인 체계
- **되돌리기** — AI가 잘못 고쳤을 때 이전 상태로

네 개를 알면 Claude Code를 쉽게 쓸 수 있습니다.

## 세션 — 한 대화의 수명 주기

<SessionLifecycle />

Claude Code에서 **세션(Session)**은 하나의 대화 단위입니다. 세션 안에서 주고받은 모든 대화가 AI의 Context에 쌓이고, 세션을 어떻게 관리하느냐에 따라 응답 품질과 사용량이 달라집니다.

### 새 세션·이어받기·종료

기본 흐름은 세 가지입니다.

```shell
claude              # 새 세션 시작
claude --continue   # 마지막 세션을 바로 이어받기 (-c)
claude --resume     # 이전 세션 목록에서 선택 (-r)
```

세션 안에서는 슬래시 명령으로 흐름을 바꿉니다.

- `/exit` (또는 `Ctrl+D`) — 세션 종료
- `/help` — 쓸 수 있는 모든 명령어 목록
- `/clear`, `/compact` — 아래에서 따로 다룹니다

### /clear와 /compact — Context 비우는 두 갈래

대화가 길어지면 Context가 가득 차 응답 품질이 떨어지거나 사용량 한도에 걸립니다. 이때 두 가지 선택이 있습니다.

- **`/clear`** — 대화를 **통째로 비움**. AI는 새로 시작한 것처럼 이전 내용을 모릅니다
- **`/compact`** — 대화를 **요약해 압축**. 토큰은 줄지만 요약 과정에서 미묘한 뉘앙스와 결정의 이유가 사라지기 쉽습니다

**`/clear` 를 기본으로 삼는 편이 안전합니다.** `/compact` 는 토큰을 아낄 수 있지만 요약이 무엇을 버리는지 예측하기 어려워 결과 품질이 은근히 떨어질 때가 많습니다. 맥락을 다시 주고 싶다면 `/clear` 뒤에 필요한 파일만 `@` 로 첨부하는 쪽이 더 깔끔합니다.

## 특수 입력 — !, @, / 로 모드 전환

Claude Code 입력창의 첫 문자는 **모드 전환 스위치**입니다. 세 가지 특수 접두어가 각각 다른 동작을 트리거합니다.

| 접두어 | 모드 | 동작 |
|---|---|---|
| `!` | Bash | Claude를 거치지 않고 쉘 명령어를 직접 실행 |
| `@` | 파일 참조 | 파일을 AI의 Context에 포함 (탭 자동완성 지원) |
| `/` | 명령어 | Slash command · Custom command 호출 |

### ! — 쉘 명령 바로 실행

> ! git status

현재 작업 디렉터리에서 `git status` 를 실행합니다. 빠른 상태 확인처럼 Claude를 경유할 필요 없는 명령에 유용합니다.

### @ — 파일을 Context에 첨부

> @src/index.ts이 파일에서 에러 처리를 개선해줘

파일 경로 자리에서 탭 자동완성이 동작합니다. 언급한 파일이 AI의 Context로 직접 들어갑니다.

### / — Slash command 호출

`/model`, `/clear`, `/permissions`, `/help` 같은 내장 명령을 호출합니다. 프로젝트에 Custom Command가 정의되어 있으면 같은 방식으로 호출할 수 있습니다. Custom Command는 Chapter 06에서 자세히 다룹니다.

### 여러 줄(Ctrl+J)·이미지(Ctrl+V)

| 단축키 | 동작 |
|---|---|
| `Ctrl+C` | AI 응답 중단 |
| `Shift+Tab` | 권한 모드 순환 (아래 권한 섹션에서 설명) |
| `Ctrl+J` | 여러 줄 입력 |
| `Ctrl+V` | 이미지 붙여넣기 |
| `Esc` + `Esc` | Rewind — 마지막 AI 행동 되돌리기 |

`Enter` 는 메시지를 **바로 전송** 합니다. 긴 프롬프트를 여러 줄로 쓸 땐 `Ctrl+J` 를 쓰는 편이 안정적입니다. `\` + `Enter` 조합도 동작하지만 `Ctrl+J` 가 공식 권장입니다.

## 권한 — AI가 어디까지 움직일지 통제

<PermissionFlow />

Claude Code는 파일 수정이나 쉘 명령 실행 전에 승인을 요청합니다. 얼마나 자주 물을지, 어떤 행동을 자동 허용할지 **권한 모드**로 조절합니다.

### 매 동작마다 묻는 기본 흐름

기본 모드는 AI가 외부에 영향을 주는 행동을 시도할 때마다 묻습니다.

- **Yes (y)** — 이번 한 번만 허용
- **Always allow (a)** — 이 도구는 앞으로도 자동 허용. `settings.json` 에 저장
- **No (n)** — 거부. AI가 다른 방법을 찾거나 다음 지시를 기다립니다

`Always allow` 로 쌓인 도구 목록은 `/permissions` 명령으로 확인·제거할 수 있습니다. 실수로 잘못 허용한 도구가 있으면 여기서 되돌립니다.

### Shift+Tab — 6가지 권한 모드 순환

`Shift+Tab` 을 누르면 권한 모드가 순환합니다.

| 모드 | 의미 |
|---|---|
| `default` | 매번 확인 (기본값) |
| `acceptEdits` | 파일 편집은 자동 허용, 나머지는 확인 |
| `plan` | 코드 수정 없이 계획만 제안 (Chapter 04에서 상세) |
| `auto` | 분류기가 매 행동을 사전 검토해 자동 실행 |
| `dontAsk` | 허용된 도구는 묻지 않고 실행 |
| `bypassPermissions` | 모든 권한 확인 건너뜀 |

화면 아래 인디케이터로 현재 모드가 표시됩니다.

### Auto mode — 분류기가 매 행동을 사전 검토한다

`auto` 는 다른 자동 모드와 결이 다릅니다. `dontAsk`·`bypassPermissions` 는 허용 규칙대로 "묻지 않고 실행" 하지만, `auto` 는 **분류기(classifier) 모델**이 매 행동을 먼저 읽고 "이건 안전 / 이건 위험"을 판단합니다. 공항의 보안 검색대처럼 위험한 것만 골라 멈추는 구조입니다.

- 기본 허용 — 로컬 파일 작업, 의존성 설치, 읽기 전용 HTTP, 시작 브랜치로의 push
- 기본 차단 — `curl | bash`, 시크릿 유출, prod 배포, `main` 에 force push, 대량 삭제

활성화는 `Shift+Tab` 순환 또는 시작 시 플래그로 지정합니다.

```shell
claude --permission-mode auto
```

**요건**: Opus 또는 Sonnet 모델, Team·Enterprise·Max 플랜. Pro 플랜과 Haiku 모델에서는 사용 불가.

<Callout type="warn" title="Auto는 편의지 면죄부가 아닙니다">
현재 Research Preview 상태입니다. 분류기가 **대부분**의 위험 행동을 걸러내지만 100% 는 아닙니다. 검증되지 않은 스크립트나 prod 인프라 작업에는 여전히 기본 모드를 쓰는 편이 안전합니다.
</Callout>

### --dangerously-skip-permissions — 분류기도 없는 마지막 수단

```shell
claude --dangerously-skip-permissions
```

모든 권한 확인 + 분류기까지 건너뜁니다. `auto` 와 달리 **안전 필터가 전혀 없습니다**. 격리된 컨테이너나 버려도 되는 임시 디렉터리에서만 쓰기를 권장합니다.

## 실수 되돌리기 — Rewind와 Checkpoint

### Esc+Esc 또는 /rewind

AI가 엉뚱하게 고쳤을 때 `Esc` 를 빠르게 두 번 누르면 **Rewind**가 작동합니다. AI가 수행한 마지막 행동을 통째로 되돌립니다. 키 조합이 낯설면 동등한 슬래시 명령 `/rewind` 도 쓸 수 있습니다.

### Checkpoint — 여러 단계 전으로

AI가 연달아 여러 단계를 진행한 뒤라면, Rewind 창에서 **Checkpoint**를 선택해 원하는 시점까지 한 번에 되돌릴 수 있습니다. 작업 중간중간 자동으로 체크포인트가 찍혀 있어 특정 상태로 돌아갈 수 있습니다.

## 핵심 포인트 정리

1. **세션 = 한 대화 단위**: `--continue` / `--resume` 으로 이어받고, Context가 꽉 차면 `/clear` 로 비웁니다 (`/compact` 는 요약 과정에서 뉘앙스가 유실되기 쉬워 되도록 피하는 편)
2. **`!` `@` `/` 는 모드 전환 스위치**: 긴 입력은 `Ctrl+J`, 이미지는 `Ctrl+V` 로 붙입니다
3. **권한 모드는 여섯 가지**: `Shift+Tab` 으로 순환합니다. `auto` 는 분류기가 있는 자동 모드이고, `bypassPermissions` 는 안전장치 없는 마지막 수단입니다
4. **Rewind는 AI 행동 단위 undo**: `Esc+Esc` 또는 `/rewind`. Checkpoint로 여러 단계 전으로도 이동합니다

## FAQ

- **Q: `/clear` 와 `/compact` 중 뭘 써야 하나요?**
  - A: `/clear` 를 기본으로 삼으세요. `/compact` 는 요약 과정에서 미묘한 지시나 결정의 이유가 사라지기 쉬워 결과 품질이 은근히 떨어질 수 있습니다. 맥락을 다시 주고 싶다면 `/clear` 뒤에 필요한 파일만 `@` 로 첨부하는 쪽이 더 안전합니다

- **Q: Auto와 `--dangerously-skip-permissions` 중 어느 쪽이 더 안전한가요?**
  - A: Auto 쪽이 명백히 안전합니다. Auto는 분류기가 매 행동을 사전 검토하지만, `--dangerously-skip-permissions` 는 검토 없이 전부 실행합니다. 자동 흐름이 필요하면 Auto를 먼저 고려하세요

- **Q: Pro 요금제에서 Auto mode를 쓸 수 있나요?**
  - A: 현재는 Team·Enterprise·Max 플랜에서만 제공됩니다. Pro에서는 `default`·`acceptEdits`·`plan` 모드까지 사용 가능합니다

- **Q: 뭔가 환경이 이상할 땐 어떻게 점검하나요?**
  - A: 세션 안에서 `/doctor` 를 실행하면 설치·권한·연결 상태를 자동 진단합니다. 발견된 문제에는 `f` 키로 바로 자동 수정이 제안됩니다

- **Q: Rewind로 되돌린 뒤 다시 원래 상태로 돌아갈 수 있나요?**
  - A: Rewind는 한 방향입니다. 되돌린 뒤에는 새 지시로 다른 방향을 진행하는 게 일반적입니다
