### 5.2 경쟁사 강점

1. **압도적 콘텐츠 볼륨**: 153강, 15시간 26분 -- 온라인 자기 주도 학습에 최적화된 잘게 쪼갠 구성
2. **넓은 대상**: 터미널 기초부터 시작하여 비개발자도 접근 가능
3. **다수 실전 프로젝트**: 4개 프로젝트로 다양한 시나리오 경험 제공
4. **Cursor AI 연동**: 시장 수요가 높은 IDE 통합 다룸
5. **외부 도구 생태계**: Shrimp Task Manager, Context7, Sequential Thinking 등 실무 도구 소개
6. **배포 워크플로우**: GitHub Pages, Vercel 배포까지 포함
7. **PRD/ROADMAP/MVP**: 업계 표준 용어로 워크플로우 설명
8. **Supabase 풀스택**: 데이터베이스 연동 프로젝트로 현실감 있는 앱 제작
9. **지속적 업데이트**: Claude Code 업데이트에 맞춰 콘텐츠 추가
10. **커뮤니티 기반**: YouTube 27K+ 구독자, 인프런 Q&A 활성화

### 5.3 자체 강의 강점 (차별화 요소)

1. **이론적 깊이**: LLM 작동 원리부터 설명 (Hallucination이 왜 발생하는지, Tool Use가 뭘 해결하는지)
2. **교육학 기반 설계**: Bottom-up, Use-Discover-Extend 구조
3. **SDD 방법론**: 경쟁사에 없는 독자적 방법론 (EARS 형식, 적합성 테스트 원칙)
4. **Context 관리 철학**: Context Rot 연구, Smart/Dumb Zone, 지침의 저주 -- 연구 근거 기반
5. **What vs How 프레임워크**: AI 위임의 본질적 사고법 제공
6. **풍부한 비유**: 레스토랑 주방, 회의실 등 기억에 남는 비유 체계
7. **병렬 에이전트 리팩토링**: 대규모 코드베이스 다루는 고급 기법
8. **MCP 개발**: 단순 사용이 아닌 직접 구축 역량
9. **학술적 레퍼런스**: 모든 개념에 출처와 근거 제시
10. **"왜"에 대한 답변**: 단순 사용법이 아니라 각 도구가 존재하는 이유와 원리 설명

## 6. 온라인 강의 전환 시 개선 권장사항

### 6.1 즉시 반영 (필수)

#### A. 콘텐츠 분할
- 현재 2시간 단위 챕터를 **5-15분 단위 영상**으로 분할 (청킹 원칙)
- 각 영상 = 1개 개념 + 1개 데모/실습
- 153강 이상은 필요 없지만, 최소 **60-80강 구성** 권장

#### B. 프로젝트 다양화
- 현재 Todo 앱 1개 -> 최소 **3개 프로젝트**로 확장
- 권장 프로젝트 구성:
  - **Project 1 (Quick Win)**: 포트폴리오 사이트 + 배포 (1-2시간, 성취감 빠르게)
  - **Project 2 (핵심)**: Todo/Kanban 앱 + SDD 워크플로우 (기존 강의 핵심)
  - **Project 3 (풀스택)**: Supabase 연동 앱 + 커스텀 MCP (심화)

#### C. 배포 워크플로우 추가
- Vercel 배포 (Next.js 프로젝트)
- GitHub Pages (정적 사이트)
- 수강생이 결과물을 공유할 수 있어야 동기 부여

#### D. Cursor AI 섹션 추가
- 전용 섹션은 아니더라도 **보조 도구로 Cursor 활용법** 1-2강 추가
- Claude Code + Cursor 조합 워크플로우 시연
- 시장 수요 반영 (경쟁사도 "with Cursor AI" 부제 사용)

### 6.2 차별화 강화 (권장)

#### E. SDD를 더 전면에
- 경쟁사는 PRD/ROADMAP/MVP 워크플로우 사용
- 자체 강의의 SDD가 더 체계적 -> 이를 핵심 USP(고유 가치 제안)로 포지셔닝
- "Design -> Plan -> Task -> Implementation" 사이클을 강의 전체 관통하는 스토리라인으로

#### F. Context 관리 심화를 독립 모듈로
- Context Rot, Smart/Dumb Zone, 지침의 저주 -> 경쟁사에 없는 내용
- 이 내용을 **독립된 "Context 마스터" 모듈**로 분리하여 마케팅 포인트로 활용
- "다른 강의에서는 배울 수 없는 컨텍스트 최적화 전략"

#### G. What vs How 프레임워크 강조
- AI 시대 개발자 사고법의 전환을 다루는 유일한 강의로 포지셔닝
- 테스트 기반 위임, 검증 가능한 목표 설정 -> 실무에 바로 적용 가능한 차별점

### 6.3 추가 콘텐츠 (선택)

#### H. 외부 도구 생태계 소개
- Shrimp Task Manager, TaskMaster 등 소개 (1-2강)
- Skills.sh 커뮤니티 활용법
- 단, 도구 종속이 아닌 **원리 이해 후 도구 선택** 관점 유지

#### I. 비개발자 진입 경로
- 터미널 기초 보충 영상 (선택 시청)
- 경쟁사가 비개발자까지 흡수하는 구조를 참고하되, 자체 강의는 **"개발자를 위한"** 포지셔닝 유지

#### J. Supabase / 데이터베이스 프로젝트
- 풀스택 프로젝트의 현실감을 위해 Supabase MCP 연동 프로젝트 추가 고려
- 이미 vault에 Supabase 강의 자료가 있으므로 활용 가능
