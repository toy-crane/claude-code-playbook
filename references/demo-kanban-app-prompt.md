# Demo Kanban App - One-shot Prompt

## 용도

Lesson 03 "Agentic 코딩과 Claude Code"의 **[데모] 같은 프롬프트, 다른 결과** 시연용.
동일한 프롬프트를 Tab A(빈 환경)와 Tab B(하네스 세팅 환경)에 동시 입력하여, 복잡한 앱일수록 하네스 유무가 결과 품질에 극적인 차이를 만든다는 것을 보여준다.

- **Tab A**: Claude Code만 설치한 빈 디렉토리
- **Tab B**: CLAUDE.md, 테스트, 컴포넌트 구조 등 하네스가 세팅된 환경

## 기술 스택
- Next.js (App Router)
- TypeScript
- shadcn/ui
- Vitest + React Testing Library
- 데이터 저장: localStorage (DB 없음)

## 칸반 보드

3개의 고정 칼럼으로 구성된다:
- Todo: 새로 추가된 카드가 여기에 들어간다
- In Progress: 진행 중인 작업
- Done: 완료된 작업

카드를 드래그&드롭으로 칼럼 사이에 이동할 수 있다.

## 카드

각 카드에는 다음 정보가 포함된다:
- 제목 (필수)
- 우선순위: High / Medium / Low (색상으로 구분)
- 태그: 카드에 태그를 추가할 수 있다 (예: "디자인", "버그", "기능")

카드 추가, 삭제, 인라인 편집이 가능하다.
빈 제목으로는 카드를 추가할 수 없다.

## 검색과 필터

- 카드 제목으로 검색할 수 있다
- 우선순위별, 태그별로 필터링할 수 있다
- 필터가 적용되면 해당하지 않는 카드는 숨겨진다

## 다크모드

- 라이트/다크 모드를 토글할 수 있다
- 선택한 모드는 새로고침 후에도 유지된다

## 데이터 유지

- 모든 카드 데이터는 localStorage에 저장된다
- 페이지 새로고침 후에도 보드 상태가 유지된다

- 카드의 우선순위가 색상으로 한눈에 구분되어야 한다

## 데모 포인트

이 프롬프트는 의도적으로 **유저 관점 요구사항만** 포함한다:
- 컴포넌트 구조, 상태 관리 방식, 드래그 라이브러리 선택 등은 지시하지 않음
- 테스트 프레임워크는 지정하되, 어떤 테스트를 작성할지는 지시하지 않음
- CLAUDE.md 내용 없음

이 빈 공간이 Tab A와 Tab B의 차이를 만든다. Tab B의 하네스(CLAUDE.md, 테스트, 규칙 등)가 이 빈 공간을 채워서 일관된 결과를 유도한다.
