# Showcase Service Design

## 한 문장 정의

"같이 만든 우리" — Claude Code 강의 수강생들의 프로젝트 갤러리

## 핵심 원칙

- 등록 장벽 최소화 (폼 하나, 인증 없음, 5분 내 완료)
- 결과물이 올리는 순간 예뻐 보여야 함
- 개인정보 보호 (닉네임만 공개, 이메일은 DB에만 저장)

## 목적

- 수강생 간 자극/동기부여 — 서로의 결과물을 보고 영감
- 강의 홍보/포트폴리오 — 외부에 "이런 걸 만들 수 있다"를 보여줌

## 기술 스택

- Next.js + Supabase + Vercel (강의 tech stack과 동일)
- 인증 없음 — Supabase anon key로 insert만 허용하는 RLS 정책

## 페이지 구성

| 페이지 | 역할 |
|--------|------|
| 갤러리 (메인) | 프로젝트 카드 그리드 |
| 상세 페이지 | 프로젝트 전체 정보 |
| 등록 폼 | 프로젝트 제출 |

## 카드 구성 (갤러리)

미니멀 카드로 갤러리를 깔끔하게 유지.

- 썸네일 이미지
- 프로젝트 제목
- 닉네임

## 상세 페이지 구성

클릭 시 풍성한 정보를 보여줌.

- 스크린샷 (여러 장)
- 제목 + 전체 설명
- 기능 목록
- 데모 링크
- 닉네임

## 등록 폼 필드

| 필드 | 필수 | 공개 | 비고 |
|------|------|------|------|
| 프로젝트 제목 | O | O | |
| 설명 | O | O | 자유 길이 |
| 데모 URL | O | O | 배포된 서비스 주소 |
| 스크린샷 | O | O | 복수 업로드 |
| 닉네임 | O | O | |
| 이메일 | O | X | 상품 연락용, 비공개 |

## 좋아요/투표

- 서비스 내 좋아요 기능 없음 — Discord에서 처리
- MVP 범위를 갤러리 + 등록에 집중

## 안 하는 것

- 로그인/인증
- GitHub 연동
- Deploy 기능 (원클릭 배포)
- 코드 프리뷰
- 좋아요/투표 기능 (Discord로 대체)

## 활용 계획

- 강의 전 미리 구축
- Part 3 프로젝트 완성 후 수강생이 직접 등록
- Discord에 프로젝트 공유 → 좋아요로 인기 투표
- 상품 지급 시 이메일로 연락

## 레퍼런스

- Devpost (해커톤 프로젝트 제출/갤러리)
- Vercel Templates (카드 UI 참고)
- Playbooks Skills (디렉토리 스타일 참고)
